UX / UI Requirements
Layout

Left Sidebar: Room list with search, filter (unread, favorites), new room button, each room shows avatar, name, last message, unread badge, and timestamp. Collapsible on mobile.

Main Chat Window:

Header: room name, members count, admin action menu (edit room, leave, delete), theme toggle, search bar for conversation-level search.

Messages area: scrollable container with date separators, continuous lazy loading (cursor-based) when scrolling up, show message reactions, translations (per-user), and read receipts.

Composer: text input with attachments (image/file), speech-to-text button, emoji picker, send button. Support multiline input with Cmd/Ctrl+Enter to send.

Right Info Panel: Room details, members list with role badges, pending join requests (admin view), pinned messages, shared files gallery. Collapsible on mobile.

Floating Actions: New message button on mobile; quick actions for upload and GIF/emoji.

Visual & Interaction

Use subtle elevation (soft shadows) and rounded corners (rounded-2xl) for message cards.

Messages from current user aligned right, others left. Different background/outline for translated messages.

Smooth animations with small delays for: message arrival (fade + slide), attachment previews, typing indicator (animated dots).

Buttons: primary (filled yellow), secondary (outline), ghost (transparent). Ensure strong contrast in both themes.

Loading states: skeleton loaders for rooms and messages, spinner for network actions.

Confirmation modals for destructive actions (delete message/leave room), with undo snackbar for message deletion (5s undo).

Accessibility

Keyboard navigation (tab order) and aria labels for all interactive elements.

High-contrast focus styles.

Screen-reader friendly announcements for incoming messages and room changes.
